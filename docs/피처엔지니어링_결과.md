스크립트: `training/phase3_features.py`

## 최종 피처 목록과 근거

| 피처 | 근거 |
|---|---|
| `totalFloorAr` (전용면적) | Phase 1에서 타겟과 상관 0.33로 가장 강한 연속형 피처 |
| `buildYear_imputed` (건축연도, 결측 대체) | 상관 0.27. 결측 10.5%는 train 중앙값(**2003년**)으로 대체 |

| `building_age` (=계약연도−건축연도) | 파생 피처. 건축연도 자체보다 "연식"이 직관적으로 더 잘 맞을 수 있어 병행 투입 |
| `area_band` (원룸/투룸/쓰리룸+) | 국내 임대 관행 기준(0~20/20~40/40㎡~)으로 구간화. **데이터로 적합한 컷오프가 아니라 도메인 관행값임을 명시** |
| `time_index` (계약시점) | Phase 1에서 상관은 약했지만(-0.08), 시간 기반 분할과 일관되게 시계열 정보를 모델에 제공 |
| `is_jeonse` (전세 여부) | Phase 1에서 전세/월세가 평균 6.8배 차이 나는 가장 강력한 구조적 피처로 확인됨 |
| `dong_target_enc` (법정동 Target Encoding) | 아래 별도 설명 |
| `dong_onehot × 54` (법정동 One-hot) | Target Encoding과 Phase 5에서 실제 성능으로 비교하기 위해 둘 다 준비 |

**층(floor)은 Phase 1에서 원본 데이터에 아예 없는 것으로 확인되어 피처에서 제외.**

## 법정동 인코딩 — Target Encoding 설계

`(구, 동)` 조합별 로그 타겟 평균을 쓰되, 표본이 적은 동은 구 평균 쪽으로 축소(shrinkage)하는 James-Stein류 공식을 사용:

```
encoded = (n_dong × mean_dong + k × mean_sigungu) / (n_dong + k)     (k=20)
```

- **train에서만 계산**하고 val/test에는 그대로 적용 (누수 방지)
- train 기준 동 개수 54개(전체 55개 중 1개는 train 시기에 표본이 없어서 아예 안 잡힘)
- val에서 train에 없던 (구,동) 조합 0개, test에서 1개 → **구 평균으로 자동 fallback** (에러 없이 처리됨 확인)

One-hot은 train에 등장한 54개 동을 기준으로 컬럼을 고정하고, val/test에서 새로 나타나는 동은 전부 0으로 처리(=미지 카테고리를 안전하게 무시). Target Encoding과 One-hot 중 어느 쪽이 더 나은지는 실제 모델 성능(Phase 5)으로 비교 예정 — 이 단계에서는 둘 다 준비만 해둔다.

## 스케일링

이 단계에서는 스케일링을 적용하지 않았다. 트리 기반 모델(LightGBM/XGBoost/CatBoost/QRF)은 스케일링이 불필요하고, 선형모델(Linear QR)에는 Phase 5에서 그 모델 학습 직전에만 표준화를 적용할 계획 — 데이터 자체에 미리 스케일링을 박아두면 트리 모델 쪽에 불필요한 변형을 주게 되므로 분리함.

## 결측치 처리 요약

- `buildYear` 10.5% 결측 → train 중앙값 대체 + 결측 플래그
- 나머지 사용 피처(`totalFloorAr`, `time_index`, `is_jeonse`, 지역)는 결측 없음 (Phase 1 확인)
