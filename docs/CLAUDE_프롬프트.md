## 1. 프로젝트 배경 (읽고 이해할 것)

### 문제

**다가구주택은 건물 전체가 등기 1개다.** (아파트처럼 호실별 구분등기가 아님)

그래서:
- 근저당은 은행이 **등기**하므로 등기부에 보임
- 임차인 보증금은 **확정일자만 받고 등기 안 함** → **등기부에 안 보임**

9가구 다가구에 입주하는 청년은, **앞선 8명이 총 얼마를 걸고 있는지 모른 채** 계약해야 한다.

### 실측 검증 (이미 완료)

직방 '지킴진단 리포트'를 실제 발급받아 확인함.

- 대상: 수원 팔달구, **일반건물(주택 9가구)**, 월세 500/40
- 결과: `매물진단 양호 / 집주인진단 양호 / 근저당 없음 / 압류 없음`
- AI 종합의견: "보증금 대비 기존 채무 비율은 안정적인 편"

**→ 9가구 건물인데, 선순위 임차인 얘기가 리포트에 한 줄도 없음.**

### 경쟁 서비스 분석 결과

| 서비스 | AI 실체 |
|---|---|
| 직방 지킴진단 | 데이터 조회 + 규칙 판단 + **LLM 요약**. 예측 모델 없음. 6~7단계는 **사람(전문가)**이 함 |
| 서울시 내집스캔 | KCB 신용점수 가져다 씀 + 룰 엔진 |
| 안심전세앱 | 데이터 통합 + 룰 엔진 |

**공통점: 아무도 "미래 사고 확률"을 예측하지 않는다.**
이유는 **라벨(사고 데이터)이 없어서**다.

### 확정일자 부여현황으로 해결 안 되는 이유

- **계약 전에는 열람 불가** (임대인 동의 필요) → 정작 의사결정 시점에 못 봄
- 확정일자 미신고 임차인은 **누락**
- **학습 데이터로 쓸 수 없음**: 임차인 본인만 열람 가능 → 라벨 수집 원천 불가

**→ 그래서 학습이 아니라 시뮬레이션으로 접근한다. 이건 제약이 아니라 방법론적 필연.**

### [A] 숨은 선순위 보증금 분포 추정 ← 지금 검증 중인 부분

**목표:** 등기부에 없는 선순위 임차보증금 총액을 **확률분포**로 추정

**입력:**
| 데이터 | 출처 | 용도 |
|---|---|---|
| 가구수 N | 건축물대장 표제부 | 선순위 임차인 = N-1명 |
| 호별 면적 | 건축물대장 전유공용면적 | 세대별 보증금 규모 |
| 전월세 실거래 | 단독/다가구 실거래가 API | 지역·면적별 보증금 분포 |

**알고리즘 (Monte Carlo):**
```python
# 1. 지역·면적대별 실거래 분포 추정
f_deposit = KDE(실거래[팔달구, 면적대])
p_jeonse  = 전세계약수 / 전체계약수

# 2. 10,000회 시뮬레이션
for i in range(10000):
    총선순위 = 0
    for j in range(N-1):                       # 앞선 세대들
        is_jeonse = Bernoulli(p_jeonse)
        보증금    = sample(f_deposit | is_jeonse, 면적_j)
        총선순위 += 보증금
    results.append(총선순위)

# 3. 출력: 평균, 90% 신뢰구간, 상위 5% (최악)
```

### [B] 경매 배당 시뮬레이션

**목표:** 사고 시 내 보증금 **실제 회수액**을 분포로 산출

**배당 순위:**
```
1순위  경매비용
2순위  최우선변제금 (소액임차인, 지역별 한도)
3순위  확정일자·근저당 설정일 순 (시간 순위)
```

```python
for i in range(10000):
    낙찰가   = 시세 × sample(낙찰가율_분포)   # 법원경매 통계
    선순위   = sample(A모듈_결과분포)          # ← A와 연결
    
    배당가능  = 낙찰가 - 경매비용
    배당가능 -= 최우선변제_총액
    배당가능 -= 근저당                        # 등기부
    배당가능 -= 선순위                        # ← 핵심
    
    내회수액  = clip(배당가능, 0, 내보증금)
```

**출력:** `전액회수 X% / 일부회수 Y% / 전액손실 Z% / 기대회수액 W원`

### [C] 사고 확률 예측 — PU Learning

**왜 아무도 안 하는가: 라벨이 없어서.**

```
일반 지도학습:  사고 O  vs  사고 X      → 학습 가능
현실:          사고 O  vs  알 수 없음  → Negative 부재
```

**Positive-Unlabeled Learning:**

| 구분 | 데이터 |
|---|---|
| Positive | **HUG 악성임대인 명단** (안심전세포털 공개) |
| Unlabeled | 나머지 전체 다가구 |

**★ 핵심 피처 (우리만 만들 수 있는 것):**

| 피처 | 기존 서비스 | 본 프로젝트 |
|---|---|---|
| 부채비율 | 근저당 / 시세 | — |
| **실질부채비율** | ❌ 불가능 | **(근저당 + A모듈_추정선순위) / 시세** |
| 전세가율 | O | O |
| 임대인 보유물건수 | O | O |
| 지역 사고율 | — | HUG 통계 |

**→ A모듈 산출물이 C모듈의 결정적 피처가 된다. 이게 프로젝트의 핵심 novelty.**

**학습:**
```
Step 1  Spy technique → Unlabeled 중 Reliable Negative 추출
Step 2  Positive vs Reliable Negative 분류기 학습
Step 3  HUG 지역별 실제 사고율로 확률 캘리브레이션
```
