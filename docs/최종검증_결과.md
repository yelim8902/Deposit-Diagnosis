## 2. 기존 KDE vs 신규 학습모델 비교

스크립트: `training/phase8_final_comparison.py` — 같은 건물(우만동 39-7), 같은 실거래 표본(373건), 같은 seed(42)로 두 방식을 나란히 실행.

| | 평균 | 90% 구간 | 상위 5% |
|---|---|---|---|
| 기존 KDE (지역·전세월세만 구분) | 2.277억 | 0.657억 ~ 4.460억 | 4.460억 |
| **신규 QRF 학습모델 (면적·연식·지역 조건화)** | **2.356억** | **0.797억 ~ 4.547억** | 4.547억 |

**평균 차이 +3.5%, 구간 폭은 비슷(기존 3.803억 vs 신규 3.750억).** 이 특정 건물(우만동, 9가구, 1996년 준공)에서는 두 방식의 차이가 크지 않다 — 이는 이 건물의 특성(면적·연식·지역)이 학습 데이터의 평균적인 범위 안에 있어서일 가능성이 높다. Phase 6에서 확인했듯, **90㎡ 이상 대형·고가 세대가 낀 건물이라면 두 방식의 차이가 이보다 훨씬 크게(신규 모델이 더 보수적으로) 벌어질 수 있다.**

**신규 모델의 실질적 장점은 이 특정 사례의 숫자 차이보다는:**
- 검증된 성능 개선(val Pinball 712.6→408.5, test 390.25 — Phase 4/5/7)
- 학습된 지도학습 모델이라는 방법론적 근거(AI Challenge 심사 대응)
- 건물마다 다른 면적·연식·지역을 실제로 반영한다는 점 (기존 KDE는 모든 다가구를 지역·전세월세만으로 뭉뚱그림)

## 3. 최종 한계 (전부 정직하게 명시)

1. **학습 데이터는 "신고된" 실거래만 포함한다.** 미신고 임차인·이면계약 등은 원천적으로 반영 불가 — 이건 어떤 방법론(KDE든 학습모델이든)을 쓰든 동일한 근본적 한계.
2. **지역·기간 커버리지 한계**: 수원시 4개 구, 2023.08~2026.07 데이터로만 학습. 다른 지역 건물에 적용 시 지역(동) 원핫이 전부 0벡터로 처리되어 정밀도가 떨어짐 (Phase 7에서 명시).
3. **90㎡ 이상 고가세대 과소예측 경향** (Phase 6 오류분석) — 대형 평수가 섞인 건물은 추정치가 보수적일 수 있음.
4. **C모듈(사고확률)은 여전히 규칙 기반**이다. HUG 악성임대인 명단(127명)이 주소 매칭이 안 돼서 PU Learning 실제 학습셋을 못 만든다는 근본 제약은 이번 A모듈 리빌드로도 해결되지 않음 — PU Learning은 여전히 "다음 단계 과제".
5. **근저당·시세는 여전히 사용자 수동 입력값**이다 (CODEF 사업자등록번호 요구, VWorld 미연동).

## 4. A모듈 리빌드 전체 요약 (Phase 0~8)

| Phase | 핵심 결과 |
|---|---|
| 0. 데이터 수집 | 수원 4개구 36개월, 49,112건 |
