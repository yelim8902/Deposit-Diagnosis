# 같은 집, 다른 결론
### 다가구주택 숨은 선순위 보증금 추정 기반 전세사기 위험 예측

**KB국민은행 제8회 AI Challenge | 주제 1. 청년 주거 금융 도우미**

---

## 1. 문제 발견 — 실측에서 시작했다

### 1.1 우리는 경쟁 서비스를 직접 써봤다

직방 '지킴진단 리포트'를 실제로 발급받았다.

**대상 매물**: 경기 수원시 팔달구, **일반건물(주택 9가구)**, 월세 500/40

**결과**:
```
매물 진단      양호
집주인 진단    양호
시세 진단      양호
근저당권       없음
압류/가압류    없음

AI 종합의견: "보증금 대비 기존 채무 비율은 안정적인 편입니다"
```

### 1.2 그런데 이 건물엔 9가구가 산다

건축물대장상 **9가구**. 내가 1가구라면, **나보다 먼저 들어온 세입자가 최대 8명**이다.

8명이 각각 3,000만 원씩만 걸었어도 **선순위 2억 4천만 원**이다.

**이 리포트 어디에도 그 얘기가 없다.**

### 1.3 왜 없는가 — 구조적 이유

| | 아파트 / 오피스텔 | **다가구주택** |
|---|---|---|
| 등기 구조 | 호실별 구분등기 | **건물 전체 1개 등기** |
| 선순위 임차인 | 없음 (1채 = 1세대) | **최대 N-1명** |
| 보증금 기재 | — | **등기부에 없음** |

근저당은 은행이 **등기**하므로 보인다.
임차보증금은 **확정일자만 받고 등기하지 않는다.** 그래서 안 보인다.

> **모든 전세사기 진단 서비스는 "등기부에 적힌 위험"만 본다.
> 다가구의 진짜 위험은 등기부에 적히지 않는다.**

### 1.4 확정일자 부여현황으로도 해결되지 않는다

| 문제 | 내용 |
|---|---|
| **계약 전 열람 불가** | 임대인 동의 필요. 정작 의사결정 시점에 못 본다 |
| **누락 위험** | 확정일자 미신고 임차인은 집계되지 않는다 |
| **체납 별도** | 국세·지방세 체납은 보증금보다 우선변제되나 별도 확인 |

**정확한 값을 알 수 없는 상태에서 계약해야 한다.**
→ 그래서 우리는 **점 추정이 아닌 확률 추정**으로 접근한다.

## 2. 접근 전략 — 검증된 파이프라인 위에 예측을 얹는다

### 2.1 기존 서비스의 AI는 무엇인가

직방 지킴진단의 실제 처리 단계를 분해하면:

| 단계 | 실제 기술 |
|---|---|
| 1. 등기부·건축물대장 조회 | API 자동 조회 |
| 2. 집주인 재무·채무 확인 | 데이터 조회 |
| 3. 시세·보증금 안전성 | 실거래가 조회 + 비율 계산 |
| 4. 치안·생활편의 | 공공데이터 조회 |
| **5. AI 내용 분석** | **LLM 요약** ← 유일한 AI |
| 6. 전문가 검토 | **사람** |
| 7. 맞춤 특약 작성 | **사람** |

**AI = 조회한 데이터를 쉬운 문장으로 바꾸는 역할.**
**예측 모델은 없다.**

내집스캔·안심전세앱도 구조는 유사하다. 데이터 통합 + 규칙 기반 판단.

### 2.2 우리 전략

선행연구 검토 결과, "전세사고 예측" 자체는 이미 시도된 바 있다 — 안선영·이상엽(2025, 주택금융연구)은 HF·HUG 보증가입 표본 45만여 건의 실제 사고 라벨로 로짓 회귀를 수행했고, 민병철(2023, 2024, 주택금융연구원)은 확률적 시뮬레이션과 ML로 깡통전세 발생확률을 추정했다. 다만 이들은 **아파트 중심**이거나(민병철은 논문에서 직접 "빌라 시장은 새로운 접근법이 필요하다"고 명시), **이미 보증에 가입한 선별 표본**, 혹은 **임대인 1인의 다주택 리스크**만을 다룬다.

> **기본 진단은 검증된 파이프라인을 따르되,
> 아무도 다루지 않은 "다가구 한 건물 내 여러 세입자가 경합하는 누적 선순위 보증금"이라는
> 구조적 정보공백을 정량 모델링해 얹는다.**

```
직방 파이프라인                     본 프로젝트
────────────────────────────    ─────────────────────────
1. 등기부·건축물대장 조회      →  ✅ 동일
2. 집주인 재무·채무            →  ✅ 동일
3. 시세·보증금 안전성          →  ⭐ [A] 선순위 분포 추정
                                  ⭐ [B] 경매 배당 시뮬레이션
4. 치안·생활편의               →  ✅ 동일
5. AI 요약                     →  ✅ 동일 (LLM)
                               →  ⭐ [C] 사고 확률 예측 (PU Learning)
                               →  ⭐ [D] 기대손실 산출
6. 전문가 검토                 →  ❌ 자동화로 대체
7. 맞춤 특약                   →  ⭐ [E] KB 금융상품 액션
8. 리포트 발급                 →  ✅ 동일
```

**에너지 배분**: 기본 진단은 빠르게 구현, **다가구 특별 진단에 전력 투입**

## 3. 핵심 모듈

### [A] 숨은 선순위 보증금 분포 추정

**해결 과제**: 등기부에 없는 선순위 임차보증금 총액 추정

**입력 데이터**
| 데이터 | 출처 | 용도 |
|---|---|---|
| 총 가구수 N | 건축물대장 API | 선순위 임차인 최대 N−1명 |
| 호별 면적 | 건축물대장 (전유부) | 세대별 보증금 규모 |
| 전월세 실거래 | 국토부 실거래가 API | 지역·면적별 보증금 분포 |

**Monte Carlo Simulation**
```python
# 1. 지역·유형·면적대별 실거래 분포 추정
f_deposit = KDE(실거래[동, '다가구', 면적대])
p_jeonse  = 전세계약수 / 전체계약수

# 2. 10,000회 시뮬레이션
for i in range(10000):
    총선순위 = 0
    for j in range(N-1):                      # 앞선 세대들
        is_jeonse = Bernoulli(p_jeonse)
        보증금    = sample(f_deposit | is_jeonse, 면적_j)
        총선순위 += 보증금
    results.append(총선순위)

# 3. 분포 산출
```

**출력**
```
선순위 보증금 총액 추정 (9가구, 수원 팔달구)
  평균        1억 8,000만 원
  90% 구간    1억 1,000만 ~ 2억 7,000만 원
  최악(상위5%) 3억 원 이상
```

**왜 학습이 아니라 시뮬레이션인가**
확정일자 부여현황은 **임차인 본인만** 열람 가능하다. 정답 라벨을 수집할 방법이 원천적으로 없다.
→ **라벨 없이 접근 가능한 유일한 방법이 시뮬레이션이다.** 이는 제약이 아니라 방법론적 필연이다.

### [B] 경매 배당 시뮬레이션

**해결 과제**: 사고 시 내 보증금 실제 회수액 산출

**배당 순위**
```
1순위  경매비용
2순위  최우선변제금 (소액임차인, 지역별 한도)
3순위  확정일자·근저당 설정일 순 (시간 순위)
```

**시뮬레이션**
```python
for i in range(10000):
    낙찰가   = 시세 × sample(낙찰가율_분포)    # 법원경매 통계
    선순위   = sample(A모듈_분포)               # ← A 결과 연결
    
    배당가능  = 낙찰가 - 경매비용
    배당가능 -= 최우선변제_총액
    배당가능 -= 근저당                          # 등기부
    배당가능 -= 선순위                          # ← 여기가 핵심
    
    내회수액  = clip(배당가능, 0, 내보증금)
```

**출력** (보증금 5,000만 기준)
```
전액 회수    42%
일부 회수    31%  (평균 2,300만)
전액 손실    27%
──────────────────────
기대 회수액  2,100만 원
```

### [C] 사고 확률 예측 — PU Learning의 아이디어를 규칙 기반으로 근사

**왜 개별 건물 단위 사고예측이 드문가**: 지도학습에 쓸 **라벨(정답 사고 데이터)이 사실상 없기 때문이다.**

```
일반 지도학습:  사고 O  vs  사고 X      → 학습 가능
현실:          사고 O  vs  알 수 없음  → Negative 부재
```

**Positive-Unlabeled Learning — 컨셉**

| 구분 | 데이터 |
|---|---|
| **Positive** | HUG 악성임대인 명단 소유 건물 (공개) |
| **Unlabeled** | 나머지 전체 다가구 (사고 여부 불명) |

**핵심 피처 — 우리만 만들 수 있는 것**

| 피처 | 기존 서비스 | 본 프로젝트 |
|---|---|---|
| 부채비율 | 근저당 / 시세 | — |
| **실질부채비율** | ❌ 불가 | **(근저당 + A모듈 추정선순위) / 시세** |
| 전세가율 | ✅ | ✅ |
| 임대인 보유물건수 | ✅ | ✅ |
| 지역 사고율 | — | HUG 통계 |

> **A모듈의 산출물이 C모듈의 결정적 피처가 된다.
> 이 피처는 다른 누구도 만들 수 없다.**

**학습 (이상형) vs 실제 구현 (현재 버전)**
```
이상형   Step 1  Spy technique → Unlabeled 중 Reliable Negative 추출
        Step 2  Positive vs Reliable Negative 분류기 학습
        Step 3  HUG 지역별·유형별 실제 사고율로 확률 캘리브레이션

현재판   HUG 악성임대인 명단(127명)은 주소·건물 단위 구조화 데이터가 없어
        개별 건물에 매칭할 방법이 없음 → 대량 Unlabeled 학습셋 구축이 비현실적.
        따라서 실제 분류기 학습 대신, 부채비율 차원은 안선영·이상엽(2025)이 HF·HUG
        실제 사고데이터 453,122건으로 추정한 로지스틱 회귀 오즈비(표10, 전부 p<0.001)를
        그대로 가져와 우리 "실질부채비율"에 적용한다(단, 논문의 부채비율은 등기부
        기준이라 실질부채비율에 적용하는 것은 근사·외삽임을 인지). 전세가율 차원은
        논문에 없는 지표라 사람이 정한 휴리스틱으로 남겨둔다. 마지막으로 HUG 공개
        전국 사고율로 확률 스케일만 캘리브레이션한다.
        → PU Learning(개별 건물 학습)은 "제안하는 다음 단계"로 명시하고, 정직하게 한계로 공개.

        ★ 흥미로운 발견: 같은 논문에서 "단독다가구" 건물유형 더미는 오즈비 0.815(음수, p<0.01)
        — 등기부 기준 부채비율을 통제하면 다가구가 오히려 위험이 낮게 나온다. 그래서 "다가구니까
        무조건 위험 가중치 추가"라는 보정은 넣지 않았다. 대신 다가구의 진짜 위험은 등기부에 잡히지
        않는 실질부채비율 자체에 있다는 본 프로젝트의 핵심 주장을, 이 논문이 역설적으로 뒷받침한다
        — 공식 지표만 보면 안전해 보이는 이유가 바로 우리가 푸는 문제이기 때문이다.
```

### [D] 기대손실 — 등급이 아닌 금액으로

```
기대손실 = P(사고) × (보증금 − E[회수액])
           ────────   ──────────────────
           [C] 출력      [B] 출력
```

**등급(양호/주의)이 아니라 원(₩) 단위로 위험을 표현한다.**
금액이어야 금융상품과 비교할 수 있기 때문이다.

---

### [E] KB 금융 액션 — 은행만 할 수 있는 것

기대손실이 **금액**이므로, 금융상품과 **직접 손익 비교**가 가능하다.

| 액션 | 비용 | 효과 | 판단 |
|---|---|---|---|
| **HUG 전세보증보험** | 연 15만 원 | 손실 0원 | 기대손실 290만 → **가입 시 순이득 275만** |
| **월세 전환** (보증금↓) | 월세 +10만 (연 120만) | 기대손실 30만으로 감소 | 손실감소 260만 > 비용 120만 → **유리** |
| **버팀목 전세자금대출** | 금리 2.1% | — | 소득요건 충족 시 |

> 직방·내집스캔은 "위험합니다"에서 끝난다.
> **KB는 "이 상품에 가입하면 275만 원 이득입니다"까지 계산한다.**

### 기술적 기여

1. **관측 불가능 변수의 확률적 추정**
   등기부에 존재하지 않는 선순위 임차보증금을 공개 데이터만으로 분포 추정

2. **라벨 부재 하의 위험 근사**
   개별 건물 단위 사고 라벨을 구할 수 없는 현실적 제약 하에서, 규칙기반 스코어링 + HUG 통계 캘리브레이션으로 근사(PU Learning은 데이터 확보 시 다음 단계로 명시)

3. **불확실성 → 금융 의사결정 연결**
   등급이 아닌 기대손실 금액 산출 → 금융상품 가입 여부를 정량적 손익으로 판단
