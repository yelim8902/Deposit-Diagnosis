# 베이스라인 결과 (Phase 4)

스크립트: `training/phase4_baselines.py` — val셋(9,609건)에서 측정. 전부 만원 단위.

## 평가지표 정의

- **Pinball Loss**: 분위수 손실의 핵심 지표. 7개 분위수(5/10/25/50/75/90/95%)에 대해 계산 후 평균.
- **CRPS 근사**: `2 × 평균 Pinball Loss`로 근사(이산적인 7개 분위수만 썼기 때문에 진짜 CRPS의 근사치임을 명시).
- **Coverage@90**: 예측 [p05, p95] 구간이 실제값을 포함하는 비율. 이상적으로는 90%에 가까워야 함.
- **Interval Width**: [p05, p95] 구간의 평균 폭 (좁을수록 좋으나, Coverage와 trade-off).
- **MAE/RMSE**: 중앙값(p50) 예측 기준, 참고용.

## 예상 밖의 발견 ★

**Baseline 3(현재 프로덕션 A모듈이 쓰는 방식 — 동/면적 조건화 없이 전세·월세만 나눠서 KDE)이 모든 지표에서 가장 좋다.** 동별(1), 동×면적별(2)으로 더 세분화할수록 오히려 성능이 나빠지는 역설적인 결과.

**원인 추정**: Baseline 1·2는 세분화할수록 그룹당 표본 수가 줄어들어(예: 특정 동×면적구간 조합은 표본이 10건 미만인 경우도 있었음 — fallback 50건 발생) 그 그룹의 경험적 분위수 추정 자체가 노이즈에 취약해진다. 반면 Baseline 3(KDE)은 전세/월세라는 큰 두 그룹으로만 나눠서 표본을 최대한 모아 부드러운 분포를 추정하기 때문에, 편향(지역·면적 미반영)은 있어도 분산이 작아 전체적으로 더 안정적인 예측이 된다 — **전형적인 편향-분산 트레이드오프**.

## 이게 Phase 5에 주는 의미

**Phase 5의 학습 모델(LightGBM Quantile 등)이 이겨야 할 진짜 기준선은 Baseline 3(Pinball 712.6)이지, 단순 세분화 베이스라인이 아니다.** 즉 "동·면적 정보를 태워서 더 정교하게 예측한다"는 것 자체는 이미 Baseline 1·2가 시도했지만 실패했다 — 관건은 **동·면적 정보를 과적합 없이 규제(regularization)하면서 반영**할 수 있느냐다. 이게 정확히 그래디언트 부스팅류 quantile 모델(적은 표본 그룹은 자동으로 덜 분할하고, 정보가 있는 곳만 세분화)이 잘하는 일이라, Phase 5에서 검증할 핵심 가설이 된다.

## Phase 5로 넘어가도 되는가?

문제 없음. **베이스라인 4개 중 최고는 Baseline 3 (Pinball 712.6, CRPS 1,425.1)** — 이 숫자를 Phase 5 모델 비교표의 기준선으로 그대로 가져간다.
