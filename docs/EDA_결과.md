## 1-2. 타겟 변수(보증금) 분석 ★

- 원본 분포: skew=2.07, kurtosis=5.33 (심하게 오른쪽 꼬리) — 분위수 q50=1,000만원, q95=18,000만원, q99=26,000만원
- **로그변환(log1p) 후: skew=0.005, kurtosis=-0.145 — 거의 정규분포에 가까워짐.**
- **전세(n=13,408) 평균 11,827만/중앙값 10,000만 vs 월세(n=35,704) 평균 1,731만/중앙값 500만** — 완전히 다른 두 집단.
- `docs/eda_plots/target_distribution.png`에서 원본·로그변환 히스토그램이 **뚜렷한 이봉분포(bimodal)** 를 보이는데, 전세/월세로 색을 나눠 보면 각각의 봉우리가 정확히 전세·월세에 대응함 — 즉 이봉분포는 **관측되지 않은 잠재변수 때문이 아니라 이미 아는 범주형 변수(전세 여부) 때문**. → **MDN처럼 잠재 다봉분포를 다루는 모델은 정당화되지 않음** (본인이 쓴 원칙 "EDA에서 실제 다봉분포 확인되면 그때 추가"와 정확히 반대 결과가 나온 것 — 여기서는 뺀다).

## 1-3. 이상치 탐지

- 보증금 0원: 106건(0.22%) — 계약상 있을 수 없는 값(데이터 오류로 판단). **→ 결정: 제거.**
- IQR 기준 이상치: 3,695건(7.52%) — 그러나 이 분포 자체가 원래 두 집단이 섞인 오른쪽 꼬리 분포라 IQR(정규분포 가정) 기준은 여기선 신뢰도가 낮음. z-score>3도 마찬가지(878건, 1.79%).
- 최댓값 81,000만원(8.1억) 등 상위값들은 실제 존재하는 고가 전세로 보이며(면적·건축연도와 함께 봤을 때 부자연스럽지 않음), 인위적 오류로 볼 근거가 없음. **→ 결정: 0원만 제거하고 나머지 "이상치"는 남긴다 — 로그변환 후 정규분포에 가까워지는 것으로 이미 확인했으므로, 굳이 IQR/z-score로 잘라내지 않고 모델(quantile regression)이 꼬리 분포를 그대로 학습하게 둔다.**

## 1-4. 피처별 관계 분석

- 전용면적 vs 보증금: r=0.33 (log 타겟 기준 0.317) — 중간 정도 양의 상관
- 건축연도 vs 보증금: r=0.27 — 최근 지어진 건물일수록 보증금 높음
- 계약시점(time_index) vs 보증금: r=-0.08 — **뚜렷한 시간 추세 없음** (월별 중앙값이 1,000~2,000만 사이에서 들쭉날쭉, 완만한 우상향/우하향 트렌드가 아니라 노이즈에 가까움 — `feature_relationships.png` 좌하단)
- 법정동별 보증금: 동별 편차가 매우 큼 (이의동·신동은 중앙값 1억 이상, 매탄동·원천동·세류동은 훨씬 낮음) — **지역이 강한 피처**
- 상관행렬: 피처 간(면적·건축연도·시점·월세) 상호 상관은 전부 |r|<0.13으로 낮음 → **다중공선성 문제 없음**. monthlyRent와 deposit은 -0.57로 강한 음의 상관인데, 이는 전세/월세 구분 자체가 만드는 구조적 관계(월세 0이면 전세)라 예상된 결과.

## 1-5. 카디널리티 / 희소성

- 법정동 고유값 55개, 동별 표본 수 중앙값 314건
- **표본 10건 미만 동 4개**: 입북동(8) / 중동(3) / 영동(1) / 상광교동(1) — 전체 55개 중 4개뿐이라 관리 가능한 수준. **→ 결정: 이 4개 동은 Target Encoding 시 구(sigungu) 단위 평균으로 fallback(shrinkage) 처리.**

## 반드시 답할 질문 — 답변

1. **타겟 분포 어떻게 생겼나? 로그변환 필요한가?** → 심한 우측 꼬리 + 이봉(전세/월세 혼재). **로그변환 필요, 그리고 전세/월세를 나누면 이봉분포가 해소됨.**
2. **전세/월세를 하나의 모델로? 분리?** → 평균이 6.8배 차이 나는 완전히 다른 두 집단. **`is_jeonse`를 강한 피처로 넣거나, 아예 두 모델로 분리하는 것을 Phase 3~5에서 둘 다 시도해서 비교.** (일단 하나의 모델 + is_jeonse 피처로 시작, 성능 부족하면 분리)
3. **시간 추세 있는가? 시간 기반 분할 필요한가?** → 상관 -0.08로 약함, 뚜렷한 추세 없음. 다만 3년치 데이터가 있고 실제 서비스 상황(과거로 미래를 예측)과 일치시키기 위해 **그래도 시간 기반 분할을 채택** (약한 추세라도 랜덤 분할의 낙관적 편향보다 안전한 선택).
4. **어떤 피처가 강하게 연관되나?** → 전세/월세 여부(구조적), 지역(법정동), 면적(r=0.33), 건축연도(r=0.27) 순.
5. **표본 부족 지역/구간 있는가?** → 55개 동 중 4개(입북동·중동·영동·상광교동)가 10건 미만. 구 단위 fallback으로 처리.
